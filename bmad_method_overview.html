Hướng Dẫn Áp Dụng BMAD v6.2.0

Tích hợp OpenCode & OH-MY-OPENCODE (OMO). Biến AI từ một công cụ chat ngẫu hứng thành một dây chuyền sản xuất phần mềm có kỷ luật, nhất quán và hiệu quả.

Phần 1: Tổng quan BMAD & Định vị

Hệ thống > Prompt đơn lẻ | Artifacts > Giao tiếp miệng | Iteration > Generation.

Quy trình hiện tại

  • BA/PM: Viết ticket Jira trực tiếp, mô tả không có cấu trúc cố định.
  • Tech Lead: Phác thảo kiến trúc trong đầu/whiteboard, ít document. Tự break task chất lượng không đều.
  • Dev: Đọc ticket Jira → tự hỏi Copilot theo ý cá nhân → Code → Push.
  • Review: Làm thủ công, phụ thuộc vào thời gian và cảm tính của người review.

Sau khi áp dụng BMAD Bundle

  • BA/PM Agent: BA viết Brief thô (1-2 trang) → PM Agent sinh ra PRD.md chuẩn hóa.
  • Architect Agent: Sinh ra architecture.md (có LikeC4 as Code). Dùng SM Agent để tạo story-*.md có Acceptance Criteria rõ ràng.
  • Dev: Chạy /vnpt-dev-story-loop → AI tự load skill Flutter, đọc kiến trúc và triển khai code.
  • Review Agent: Chạy /vnpt-review-loop song song, lặp tự động đến khi 0 actionable issue.
Lưu ý về Công cụ: VNPT đang sử dụng MiniMax-M2.7 làm model mặc định. BMAD chạy qua OpenCode CLI. GitHub Copilot trong VS Code chỉ đóng vai trò thứ yếu (secondary).

Phần 2: Cài đặt & Thực hành Step-by-Step

Dành cho việc setup ban đầu và luồng công việc hàng ngày của Dev/Lead.

1 Cài đặt môi trường & Bundle VNPT

Yêu cầu hệ thống: Node.js (20+), Python (3.8+), Git.

Terminal - Cài đặt OpenCode
npm install -g opencode-ai opencode --version
Terminal - Clone Bundle & Cài đặt vào Project
# 1. Clone Bundle nội bộ VNPT git clone <VNPT-INTERNAL-GIT>/vnpt-bmad-custom cd vnpt-bmad-custom # 2. Chạy installer trỏ vào project Flutter của bạn (Thay đường dẫn) python install_vnpt_bmad_custom_all.py --repo /duong/dan/toi/du-an-flutter --skip-serena --skip-gitnexus

Lệnh này sẽ cài đặt BMAD v6.2.0, nạp OMO, copy custom skills (ui-ux-pro-max, springboot-patterns) và tạo folder _bmad/ trong project của bạn.

2 Các Lệnh Workflow Hàng Ngày (Mở Terminal tại thư mục Project)

A. Tạo PRD từ Brief (BA/PM)

Tạo file docs/brief-feature.md (1-2 trang mô tả thô), sau đó chạy lệnh:

Command
/bmad-create-prd docs/brief-feature.md

B. Thiết kế Kiến trúc (Lead)

Gọi Agent phân tích sâu và sinh mã LikeC4 as Code dựa trên PRD.

Command
@bmm-architect Hãy đọc docs/prd.md và tạo architecture document.

C. Chia nhỏ Task (Lead/SM)

Tạo các file story-*.md với Acceptance Criteria.

Command
@bmm-sm Dựa trên prd.md và architecture.md, chia nhỏ thành các stories.

D. Triển khai & Review (Dev)

Dev Agent sẽ tự load skill Flutter/OMO. Sau đó chạy Review loop để fix bug.

1. Code Loop
/vnpt-dev-story-loop docs/stories/story-001.md
2. Review Loop
/vnpt-review-loop
